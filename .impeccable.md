## Design Context

GAIA is a proactive personal AI assistant that handles the cross-tool busywork eating its users' days — Gmail, Calendar, Linear, Slack, Notion, WhatsApp. The product spans web, desktop (Electron), mobile (React Native), and bots across Discord/Slack/Telegram/WhatsApp. The full token system, card contract, and component rules live in [`DESIGN.md`](./DESIGN.md) — this file captures the *who*, *why*, and *feel* that DESIGN.md doesn't.

### Users

**Founders, operators, and exec-types** drowning in tool sprawl. They run companies or own outcomes — every minute spent re-explaining context to another tool is a minute stolen from real work. They reach for GAIA when their inbox, todo list, calendar, and Slack threads have become a single anxious blur and they need an assistant that already knows them.

They are technical enough to tolerate density, but they are not engineers looking for a power tool to admire. They want results, fast, with the lowest possible cognitive overhead. They are using GAIA between meetings, on a phone, late at night — usually under load.

### Brand Personality

GAIA holds four traits at once. They are not in tension when executed well; they are the brand:

- **Warm + human + thoughtful** — feels like a person who cares, not a chatbot. Copy is considered. Empty states teach. Small moments feel personal.
- **Sharp + intelligent + opinionated** — has a point of view. Surfaces decisions, doesn't ask the user to make every micro-choice. Confident defaults.
- **Calm + confident + dense** — quiet capability. Information-rich without being noisy. Earned the right to show a lot because nothing wasted space.
- **Fast + minimal + invisible** — the best moments are when the interface gets out of the way. Keyboard-driven, near-zero chrome, optimistic UI.

The emotional goal is **relief**. The user opens GAIA and feels their shoulders drop.

### Aesthetic Direction

**References to aim for:** Arc, Notion, Cron. Editorial polish. Generous space when it matters. Personality through copy and small delight moments — not through decoration. The interface is *felt* before it is read.

**Anti-references:** Generic AI-product aesthetic (cyan-on-black gradient hero, glassy purple cards, sparkline as decoration). ChatGPT-style minimalism that reads as undesigned. Dashboard maximalism with metric tiles and side-stripe borders on everything.

**Theme:** Dark-first (`#111111` primary background) — see DESIGN.md for the full token system. The user is on this product for hours, often at night, usually with focus mode on. Light mode is supported but secondary. The single brand accent is `#00bbff` cyan, and it is rare on purpose; using it twice on a screen weakens both uses.

**Typography:** Inter for the entire product surface. PP Editorial New is reserved for editorial moments — landing hero, marketing display, occasional in-product moments where the brand earns a beat. Anonymous Pro for code. The display font is the brand's "warmth" lever — use it where the product wants to feel personal, not technical.

### Design Principles

These are the calls to make when DESIGN.md doesn't have a token for your situation:

1. **Tonal layering over borders.** Depth comes from `bg-zinc-800` over `bg-zinc-900`. If you reach for `border-` or `ring-` on a card, you're solving the wrong problem. Re-stack instead.
2. **Cyan is a scarce resource.** One primary action per screen. If two things compete for the accent, neither deserves it — pick one, demote the other.
3. **Density is earned, not assumed.** Information-dense is the goal, but every element must justify its pixels. When in doubt, remove. Calm comes from restraint, not from padding.
4. **Copy is design.** Every label, every empty state, every error message is a design surface. Write it like a thoughtful colleague would say it. Cut "we", "please", and any sentence that restates the obvious.
5. **Motion is functional.** Animate state changes — entrances, exits, expansions. Never animate for delight alone. The exception: editorial/landing surfaces, where one well-orchestrated moment is allowed to exist for its own sake.
6. **Editorial moments matter.** Landing pages, onboarding, empty states, and milestone moments are where the brand earns warmth. The product is dense; these are the breathing room.
