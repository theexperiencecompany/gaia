## Prerequisites

Before you begin, ensure you have the following installed:

<AccordionGroup>
  <Accordion icon="docker" title="Docker & Docker Compose">
    - Docker Desktop or Docker Engine
    - Docker Compose
    - At least 4GB of available RAM
    - 10GB of free disk space
  </Accordion>
  <Accordion icon="code" title="Development Tools">
    - Git (for version control)
    - [mise](https://mise.jdx.dev) (task runner and tool manager)
    - Docker & Docker Compose
  </Accordion>

  <Accordion icon="terminal" title="System Requirements">

    - At least 8GB of RAM (16GB recommended)

  </Accordion>
</AccordionGroup>

## Development Environment Setup

Before running GAIA tasks, you need mise installed and activated. This section shows how to install it on different platforms and ensure it is available in your shell.

## Install mise

<Tabs>
  <Tab title="macOS/Linux (curl)" icon="terminal">
    ```bash
    curl https://mise.run | sh
    ```

    Verify installation:

    ```bash
    ~/.local/bin/mise --version
    ```
  </Tab>

  <Tab title="macOS (Homebrew)" icon="leaf">
    ```bash
    brew install mise
    ```

    ```bash
    mise --version
    ```
  </Tab>

  <Tab title="Windows" icon="windows">
    <Tabs>
      <Tab title="winget">
        ```powershell
        winget install jdx.mise
        ```
      </Tab>

      <Tab title="scoop">
        ```powershell
        scoop install mise
        ```
      </Tab>

      <Tab title="choco">
        ```powershell
        choco install mise
        ```
      </Tab>
    </Tabs>

    ```powershell
    mise --version
    ```
  </Tab>

  <Tab title="Debian/Ubuntu (apt)" icon="debian">
    <Tabs>
      <Tab title="amd64">
        ```bash
        sudo apt update -y && sudo apt install -y curl

        echo "deb [signed-by=/etc/apt/keyrings/mise-archive-keyring.pub arch=amd64] https://mise.jdx.dev/deb stable main" | sudo tee /etc/apt/sources.list.d/mise.list
        sudo apt update
        sudo apt install -y mise
        ```
      </Tab>

      <Tab title="arm64">

## Activate mise

Add mise activation to your shell so GAIA commands work.

<Tabs>
  <Tab title="bash">
    ```bash
    echo 'eval "$(mise activate bash)"' >> ~/.bashrc
    source ~/.bashrc
    ```
  </Tab>

  <Tab title="zsh">
    ```bash
    echo 'eval "$(mise activate zsh)"' >> ~/.zshrc
    source ~/.zshrc
    ```
  </Tab>

  <Tab title="fish">
    ```bash
    echo 'mise activate fish | source' >> ~/.config/fish/config.fish
    ```
  </Tab>

  <Tab title="Windows (Powershell)">
    ```powershell
    $shimPath = "$env:USERPROFILE\AppData\Local\mise\shims"
    $currentPath = [Environment]::GetEnvironmentVariable('Path','User')
    $newPath = $currentPath + ";" + $shimPath
    [Environment]::SetEnvironmentVariable('Path', $newPath, 'User')
    ```
  </Tab>
</Tabs>

## Verify mise is active

```bash
mise doctor
```
You should see that mise is activated. If not, restart your terminal.

# Setup

<Tabs>
  <Tab title="Automated Setup (Recommended)" icon="rocket">
    We use mise to automate the entire setup process. This is the recommended method for all users.


<Steps>
  <Step title="Install mise">
    Ensure mise is installed and activated before continuing.
  </Step>

  <Step title="Setup Environment Variables">
    ```bash
    cp backend/.env.example backend/.env
    cp frontend/.env.example frontend/.env
    ```
    Then configure your environment variables by following the docs.
  </Step>

  <Step title="Run Setup">
    ```bash
    mise setup
    ```

    This will:
    - check env files  
    - start docker services  
    - install Python  
    - install backend dependencies  
    - install Node 22 and frontend deps  
    - seed models  
  </Step>

  <Step title="Start Development">
    ```bash
    mise dev:backend
    mise dev:frontend
    mise dev
    mise dev:full
    ```

    See all commands with `mise tasks`.
  </Step>
</Steps>

  </Tab>


  <Tab title="Manual Setup" icon="wrench">
    If you prefer to set up your environment manually or want more control, follow these steps.


Environment Variables

<Steps>
  <Step title="Copy Example Files">
    ```bash
    cp backend/.env.example backend/.env
    cp frontend/.env.example frontend/.env
    ```
  </Step>


  <Step title="Fill in Your Secrets">
    Edit both `.env` files with your own values.
  </Step>
</Steps>


Backend Setup

<Steps>
  <Step title="Start Docker Services">
    **All services**
    ```bash
    docker-compose up -d
    ```


**Backend only**
```bash
docker compose --profile backend-only up -d
```

  </Step>


  <Step title="Create Virtual Environment">
    ```bash
    python3 -m venv .venv
    source .venv/bin/activate
    ```
  </Step>


  <Step title="Install Backend Dependencies">
    ```bash
    mise install
    ```


Or:
```bash
uv sync
```

  </Step>
</Steps>


Frontend Setup

<Steps>
  <Step title="Navigate to Frontend Directory">
    ```bash
    cd frontend
    ```
  </Step>


  <Step title="Install Frontend Dependencies">
    ```bash
    mise install
    ```


Or:
```bash
pnpm install
```

  </Step>


  <Step title="Start Development Server">
    ```bash
    mise dev
    ```
  </Step>


  <Step title="Return to Root">
    ```bash
    cd ..
    ```
  </Step>
</Steps>


  </Tab>
</Tabs>
