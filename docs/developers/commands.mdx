# Commands Reference

GAIA uses [mise](https://mise.jdx.dev) as a unified task runner for both backend and frontend development. This page provides a complete reference of all available commands.

<Tip>
  **Quick Tip:** Run `mise tasks` from any directory to see all available
  commands with descriptions.
</Tip>

## Backend Commands

Run these commands from the `backend/` directory or use `mise -C backend <task>` from the root.

### Development

<AccordionGroup>
  <Accordion icon="rocket" title="mise dev">
    **Run development server with hot reload**

    Starts the FastAPI backend with automatic reloading on code changes.

    ```bash
    mise dev
    ```

    Server will be available at `http://localhost:8000`

  </Accordion>

  <Accordion icon="play" title="mise start">
    **Run production server**

    Starts the FastAPI server in production mode.

    ```bash
    mise start
    ```

  </Accordion>

  <Accordion icon="cog" title="mise worker">
    **Start ARQ worker**

    Starts the background task worker for processing asynchronous jobs.

    ```bash
    mise worker
    ```

    Alias: `mise arq`

  </Accordion>
</AccordionGroup>

## Root Commands

Run these commands from the project root directory to manage both backend and frontend.

### Setup & Installation

<AccordionGroup>
  <Accordion icon="rocket" title="mise setup">
    **Complete project setup**

    Automated setup that handles everything:
    - Checks environment files exist
    - Starts Docker services
    - Installs Python 3.11 and Node.js 22
    - Creates Python virtual environment
    - Installs all backend dependencies
    - Installs all frontend dependencies
    - Seeds AI models and workflows

    ```bash
    mise setup
    ```

    <Warning>
      Before running setup, ensure you've copied and configured your `.env` files:
      ```bash
      cp backend/.env.example backend/.env
      cp frontend/.env.example frontend/.env
      ```
    </Warning>

  </Accordion>

  <Accordion icon="check-circle" title="mise setup:env">
    **Check if environment files exist**

    Validates that required `.env` files are present.

    ```bash
    mise setup:env
    ```

  </Accordion>

  <Accordion icon="download" title="mise install:all">
    **Install all dependencies**

    Install dependencies for both backend and frontend.

    ```bash
    mise install:all
    ```

  </Accordion>

  <Accordion icon="server" title="mise setup:backend">
    **Set up backend only**

    Install Python tools and backend dependencies.

    ```bash
    mise setup:backend
    ```

  </Accordion>

  <Accordion icon="browser" title="mise setup:frontend">
    **Set up frontend only**

    Install Node.js tools and frontend dependencies.

    ```bash

    ```

  </Accordion>

</AccordionGroup>

### Docker Management

<AccordionGroup>
  <Accordion icon="docker" title="docker compose up">
    **Start all docker services**

    Start all services with Docker Compose.

    ```bash
    docker compose up
    ```

    Run in background with `-d` flag:
    ```bash
    docker compose up -d
    ```

  </Accordion>

  <Accordion icon="stop" title="docker compose down">
    **Stop all docker services**

    Stop and remove all Docker containers.

    ```bash
    docker compose down
    ```

  </Accordion>

  <Accordion icon="file-lines" title="docker compose logs">
    **View docker logs**

    Stream logs from all Docker services.

    ```bash
    docker compose logs -f
    ```

    View logs for a specific service:
    ```bash
    docker compose logs -f postgres
    ```

  </Accordion>

  <Accordion icon="rotate" title="docker compose restart">
    **Restart all docker services**

    Restart all Docker services without recreating containers.

    ```bash
    docker compose restart
    ```

  </Accordion>

  <Accordion icon="box" title="docker compose --profile all up">
    **Start ALL containers**

    Start all containers including backend/arq in docker (profile: all).

    ```bash
    docker compose --profile all up
    ```

  </Accordion>

  <Accordion icon="box" title="docker compose --profile backend up">
    **Start docker + backend container**

    Start docker services + backend container (profile: backend).

    ```bash
    docker compose --profile backend up
    ```

  </Accordion>

  <Accordion icon="box" title="docker compose --profile worker up">
    **Start docker + worker container**

    Start docker services + ARQ worker container (profile: worker).

    ```bash
    docker compose --profile worker up
    ```

  </Accordion>

  <Accordion icon="hammer" title="docker compose build">
    **Build docker images**

    Rebuild all Docker images from scratch.

    ```bash
    docker compose build
    ```

    Build without cache:
    ```bash
    docker compose build --no-cache
    ```

  </Accordion>
</AccordionGroup>

## Need Help?

If you encounter issues with any commands:

1. Check the command is running from the correct directory
2. Ensure all dependencies are installed with `mise setup:deps`
3. Verify Docker services are running for backend tasks
4. Check the [Development Setup](/developers/development-setup) guide
