## Welcome to GAIA Self-Hosting

Self-host GAIA with Docker Compose. Keep it simple: start with minimal env vars, add more only when you enable features.

## What runs

- **Web**: Next.js
- **Backend**: FastAPI
- **Databases/Infra**: PostgreSQL, MongoDB, Redis, ChromaDB, RabbitMQ (via Docker)

## Prerequisites

- Docker Engine 20.10+ and Compose v2+
- 2+ CPU cores, 4GB+ RAM, 10GB+ disk (8GB+ RAM recommended)
- API keys as needed (see [Environment Variables](/configuration/environment-variables))

## Quick Start

<Tabs>
  <Tab title="CLI (Recommended)" icon="terminal">
    The GAIA CLI handles everything automatically â€” cloning, tools, env vars, and startup.

    ```bash
    curl -fsSL https://heygaia.io/install.sh | sh
    gaia init
    ```

    See [CLI Installation](/cli/installation) and [CLI Commands](/cli/commands) for details.
  </Tab>

  <Tab title="Manual (Docker)" icon="docker">
    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/theexperiencecompany/gaia.git
        cd gaia
        ```
      </Step>
      <Step title="Set minimal env vars">
        Add a minimal `apps/api/.env` (see the "Minimal .env to boot locally" section
        in [Environment Variables](/configuration/environment-variables)).
      </Step>
      <Step title="Start services">
        ```bash
        docker compose -f infra/docker/docker-compose.yml up -d
        ```
      </Step>
      <Step title="Open GAIA">
        - Web: http://localhost:3000
        - Backend API: http://localhost:8000
      </Step>
    </Steps>
  </Tab>
</Tabs>
