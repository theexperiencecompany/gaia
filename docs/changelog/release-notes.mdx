### [v0.16.0](https://github.com/theexperiencecompany/gaia/releases/tag/api-v0.16.0)

**Apps**: [API v0.16.0](https://github.com/theexperiencecompany/gaia/releases/tag/api-v0.16.0) · [Web v0.17.0](https://github.com/theexperiencecompany/gaia/releases/tag/web-v0.17.0) · [Mobile v0.4.0](https://github.com/theexperiencecompany/gaia/releases/tag/mobile-v0.4.0) · [Desktop v0.2.0](https://github.com/theexperiencecompany/gaia/releases/tag/desktop-v0.2.0) · [Bots v1.2.0](https://github.com/theexperiencecompany/gaia/releases/tag/bots-v1.2.0) · [CLI v0.2.1](https://github.com/theexperiencecompany/gaia/releases/tag/cli-v0.2.1)

#### Features
- **System Workflows for Gmail and Google Calendar**: Workflows are now auto-created when you connect a Gmail or Google Calendar integration, giving you ready-to-use automations out of the box
- **Desktop App Revamp**: Deep linking, branded splash screen, auto-updater, and modular window management bring the desktop experience up to parity with the web app
- **LangChain Middleware, VFS, and Skills Support**: New middleware layer for LangChain integrations, a virtual file system abstraction, and first-class skills support for agents
- **Workflow Prompt Split**: Display descriptions and execution prompts are now separate fields, giving users clearer context while preserving precise agent instructions
- **Vercel Web Analytics**: Production usage tracking via Vercel Analytics integrated into the Next.js frontend
- **Cloudflare Deployment Support**: Added Wrangler configuration, SSR chunk analysis scripts, and deploy/preview commands for Cloudflare Pages
- **Server API Base URL Centralization**: Consistent API base URL handling across all server components via a shared utility and new provider components

#### Improvements
- **Docker Image Optimization**: Slimmed down images across API, web, bots, and voice-agent with pruned unused dependencies and improved layer caching
- **LazyMotionProvider for Animations**: Navbar and main content wrapped in a lazy-loaded Framer Motion provider, reducing initial bundle size for animation-heavy pages
- **SEO Consolidation**: Crawl and index signals unified across all pages, removing SEO drift and conflicting directives
- **GCP Deployment Enhancements**: Phased image pulling and improved service verification for more reliable production deploys
- **Workflow Creation Error Handling**: Better validation and error messages when creating or updating workflows, with improved protocol registration and auth redirect support

#### Bug Fixes
- **Desktop Server Shutdown Reliability**: Fixed graceful shutdown, updated splash screen appearance, and refined login UI styling
- **Safer Protocol Registration on Linux**: Replaced execSync with spawnSync for protocol handler registration to prevent shell injection
- **Dockerfile Ownership and Permissions**: Corrected file ownership for libs directory so setuptools can create package metadata, and set proper permissions for copied files
- **Icon Consistency in Calendar**: Replaced ad-hoc icon components with the standard Chevron component in Calendar and DateTimePicker
- **Google OAuth Production Settings**: Added missing client ID and secret to ProductionSettings for Google OAuth flows

#### Infrastructure
- **Frozen Lockfile Enforcement in Docker**: Dockerfiles now use the --frozen flag for uv run commands, preventing accidental dependency drift in builds
- **Claude Code CI Workflows**: Added GitHub Actions workflows for automated code review and PR assistance powered by Claude
</Update>

### [v0.15.0](https://github.com/theexperiencecompany/gaia/releases/tag/api-v0.15.0)

**Apps**: [API v0.15.0](https://github.com/theexperiencecompany/gaia/releases/tag/api-v0.15.0) · [Web v0.16.0](https://github.com/theexperiencecompany/gaia/releases/tag/web-v0.16.0) · [Bots v1.1.0](https://github.com/theexperiencecompany/gaia/releases/tag/bots-v1.1.0) · [CLI v0.2.0](https://github.com/theexperiencecompany/gaia/releases/tag/cli-v0.2.0)

#### Features
- **GAIA Bots**: Discord, Slack, and Telegram bots ship as first-class platform integrations, letting users interact with GAIA directly from their team chat or DMs
- **Telegram bot**: Full Telegram support with serialized message updates, Markdown parsing, and streaming message breaks for long responses
- **Discord bot enhancements**: Rotating presence status, DM welcome messages, context menu commands, and resolved timeout issues
- **Bot commands**: `/help`, `/settings`, `/new`, `/stop`, and `/unlink` commands available across all bot platforms for session and account management
- **Platform linking**: Secure link-token flow connects bot accounts to GAIA user profiles, with WorkOS SSO return URL handling and a new settings UI for managing linked platforms
- **GAIA CLI**: New command-line interface (`gaia init`, `gaia status`, `gaia stop`) for self-hosting and local development, with Docker support, port conflict detection, and health verification
- **CLI initialization flow**: Guided setup with repository cloning progress, environment configuration, Docker prerequisite checks, and detailed logging at every step
- **Multi-channel notifications**: Push notifications delivered through Telegram and Discord channels in addition to the web app
- **Bot landing pages**: Dedicated landing pages for Discord and bot onboarding, with integration status displayed in bot settings
- **Streaming chat reliability**: Server-sent keepalives, client-side retries for transient errors, and adjusted timeouts keep long-running responses stable across web and bot clients
- **Bot API key authentication**: Middleware layer secures all bot-to-API communication with dedicated API keys
- **Integration status in settings**: API responses now include integration connection status, surfaced in the bot settings panel

#### Bug Fixes
- **Bot security audit**: Comprehensive fixes for deployment, security, and functional bugs across all three bot platforms
- **Bot rate limiting and mention stripping**: Public context restrictions, cleaned mention formatting, and improved rate limit UX prevent abuse and noisy responses
- **CLI lifecycle bugs**: Resolved 11 bugs across init, start, stop, and status commands, including graceful SIGINT/SIGTERM shutdown handlers and port validation (1-65535)
- **CLI memory and rendering**: Extracted log buffer sizes to shared constants, improved LogWindow React key stability, and fixed isRefreshing state resets
- **Subagent KeyError**: Fixed web_search_tool KeyError in subagent execution and OpenRouter credit handling
- **MCP OAuth**: Raise a clear error when the OAuth token is missing in MCPClient instead of failing silently

#### Infrastructure
- **Bots in Nx and CI**: Discord, Slack, and Telegram bots integrated into the Nx workspace, CI pipeline, and release system with Docker build workflows
- **Blacksmith CI runners**: All GitHub Actions workflows migrated to Blacksmith runners for faster builds
- **GHCR authentication**: Deploy workflow now authenticates with GitHub Container Registry for pulling images
- **Docker Compose fixes**: Persistent redis_data volume in production, aligned dev POSTGRES_DB naming, added RabbitMQ to selfhost depends_on, and corrected voice agent cache path
- **Centralized bot config**: Infisical secret injection and consolidated `.env.example` files across all bot apps

#### Improvements
- **Docs site overhaul**: Restructured documentation with improved styling, CLI installation page, and prioritized self-hosting guides
- **Bot streaming refactor**: Centralized streaming logic in the shared bot library with dynamic message updates and new-message-break handling
- **CLI UX overhaul**: Reordered init flow, simplified UI, deduplicated components, added `--branch` option for specifying the clone branch

#### Performance
- **Agent flow latency**: Optimized tool execution path in the agent graph to reduce end-to-end response latency
- **SSE keepalive mechanism**: Switched to JSON data events for improved client compatibility and lower dropped-connection rates
</Update>
