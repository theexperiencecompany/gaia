## Design Philosophy

GAIA's UI is **dark-first**, **flat**, and **single-accent**. Five principles govern every visual decision:

<CardGroup cols={3}>
  <Card title="Dark-First" icon="moon">
    Primary experience is dark mode (`#111111`). Light mode is supported via CSS variables but is secondary.
  </Card>
  <Card title="Flat Depth" icon="layer-group">
    Depth from layered backgrounds (`zinc-800` → `zinc-900`) only. Never borders, rings, or outlines.
  </Card>
  <Card title="Single Accent" icon="droplet">
    One primary action color: `#00bbff` cyan. Everything else is zinc-scale neutrals.
  </Card>
  <Card title="Borderless Cards" icon="square">
    Data cards use background-only separation. No `border-`, `ring-`, or `outline-` anywhere in the card tree.
  </Card>
  <Card title="Subtle Motion" icon="wand-magic-sparkles">
    Animations are functional — entrance, exit, state change only. Never decorative. ≤ 300ms for interactions.
  </Card>
</CardGroup>

### Heading Scale

Styles apply globally to semantic HTML elements — just use the right tag.

| Element | Classes | Size |
|---------|---------|------|
| `<h1>` | `text-3xl font-bold` | 30px |
| `<h2>` | `text-2xl font-bold` | 24px |
| `<h3>` | `text-xl font-bold` | 20px |
| `<h4>` | `text-lg font-bold` | 18px |
| `<h5>` | `text-base font-bold` | 16px |
| `<h6>` | `text-sm font-bold` | 14px |

### Text Patterns

**Uppercase section labels** — settings panels, card headers, form group titles:

```tsx
<p className="text-xs font-medium uppercase tracking-wider text-zinc-500">
  Section Title
</p>
```

**Truncation** — always truncate in constrained containers:

```tsx
<span className="truncate">...</span>                  // single line
<p className="line-clamp-2">...</p>                    // two lines max
<p className="line-clamp-1 max-w-[200px]">...</p>     // width + line cap
```

```tsx
  pending: "bg-zinc-700/50 text-zinc-400",
} as const;

export function MyCard({ title, items, badge }) {
  return (
    <div className="rounded-2xl bg-zinc-800 p-4 w-fit min-w-[400px]">
      <div className="flex items-center justify-between mb-3">
        <p className="text-sm font-semibold text-zinc-100">{title}</p>
        {badge && (
          <span className="rounded-full bg-zinc-700/50 px-2 py-0.5 text-xs text-zinc-400">
            {badge}
          </span>
        )}
      </div>
      <div className="space-y-2">
        {items.map((item) => (
          <div key={item.id} className="rounded-2xl bg-zinc-900 p-3">
            <div className="flex items-center justify-between gap-2">
              <span className="text-sm font-medium text-zinc-200">{item.label}</span>
              <span className={`rounded-full px-2 py-0.5 text-xs ${statusClasses[item.status]}`}>
                {item.value}
              </span>
            </div>
            {item.meta && (
              <p className="text-xs text-zinc-500 mt-1">{item.meta}</p>
            )}
          </div>
        ))}
      </div>
    </div>
  );
}
```

### Hard Rules

- `rounded-2xl` on outer containers — always
- Two-tone zinc: `zinc-800` outer → `zinc-900` inner
- Status badges always use `/10` opacity backgrounds
- **Never** `border-`, `ring-`, or `outline-` anywhere in a card tree

## Animations

### Available Classes

| Class | Duration | Easing | Use |
|-------|----------|--------|-----|
| `animate-spin` | Infinite | linear | Loading spinners |
| `animate-pulse` | Infinite | linear | Skeleton placeholders |
| `animate-accordion-down` | 0.2s | ease-out | Accordion open |
| `animate-accordion-up` | 0.2s | ease-out | Accordion close |
| `animate-scale-in` | 0.4s | bounce | Element entrance |
| `animate-scale-in-blur` | 0.5s | bounce | Blurred entrance |
| `animate-shimmer` | 2s | linear | Shimmer effect |
| `animate-shake` | 0.7s | ease-in-out | Error feedback |

### Transitions

```tsx
// Default — use everywhere
<div className="transition-all duration-200">

// Color only
<div className="transition-colors duration-200">

// Button press
<button className="active:scale-95 transition-all! duration-300">
```

### Easing Functions

| Name | Value | When |
|------|-------|------|
| Default | `ease` | Most transitions |
| Entrance / exit | `ease-out` | Entrances and exits |
| Bounce | `cubic-bezier(0.34, 1.56, 0.64, 1)` | `scale-in`, `scale-in-blur` |

### Framer Motion

Import from `motion/react` — never `framer-motion`.

```typescript
import { AnimatePresence, m } from "motion/react";

<AnimatePresence mode="wait">
  {visible && (
    <m.div
      initial={{ opacity: 0, y: 8 }}
      animate={{ opacity: 1, y: 0 }}
      exit={{ opacity: 0, y: -8 }}
      transition={{ duration: 0.2, ease: "easeOut" }}
    >
      {content}
    </m.div>
  )}
</AnimatePresence>
```

`AnimatePresence` is required for exit animations. Keep durations ≤ 300ms for interactions, ≤ 500ms for page-level entrances.

All icons come from `@icons` (`@theexperiencecompany/gaia-icons/solid-rounded`). Never use raw SVGs or Unicode symbols.

```typescript
import { CheckmarkCircle02Icon, Alert01Icon, Copy01Icon, Brain02Icon } from "@icons";
```

### Sizing

| Context | Value | Class |
|---------|-------|-------|
| Inline (badges, labels) | `height={17}` | 17px |
| Action buttons | `size={16}` | 16px |
| Prominent / decorative | `size={24}` | 24px |

### Usage Patterns

```tsx
// In a ghost button
<Button variant="ghost" size="icon">
  <Copy01Icon className="h-4 w-4" />

```

<Warning>

## Do's and Don'ts

<CardGroup cols={2}>
  <Card title="Do" icon="check">
    - Use `rounded-2xl` on all outer card containers
    - Layer `zinc-800` → `zinc-900` for card depth

    - Import icons from `@icons`
    - Use `cn()` for conditional class merging
    - Use `transition-all duration-200` as the default transition
    - Use `AnimatePresence` for exit animations with Framer Motion

    - Never add `border-`, `ring-`, or `outline-` to card trees
    - Never use `rounded-lg` on card containers
    - Never use solid backgrounds for status badges
    - Never use Unicode symbols (`→`, `•`, `✓`) in JSX — use icons
    - Never add `<Toaster>` — it's already mounted globally
    - Never use CSS variables on dark card surfaces — use zinc directly
    - Never set `font-family` inline — use Tailwind token classes
    - Never import from `framer-motion` — use `motion/react`
