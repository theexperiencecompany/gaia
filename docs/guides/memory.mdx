## What to say

Just talk. Memory works in the background, no "remember this" required.

<Chat
  messages={[
    { from: "user", text: "message sarah that I'm running late" },
    { from: "gaia", text: "Sent, I messaged Sarah on Slack that you're running late." },

    { from: "gaia", text: "Noted, 25-minute meetings are now your default." },
  ]}

If GAIA has seen Sarah's Slack handle, James's email, or your manager's name before, it fills in the rest. You don't look anything up.

## What GAIA learns

At the end of each conversation, GAIA reviews what happened and keeps anything worth remembering:

- **Identity mappings**, names linked to emails, Slack IDs, GitHub handles, calendar IDs
- **Preferences**, formats, timezones, communication style, defaults
- **Contacts and relationships**, who you message, who reports to you, key collaborators
- **Tools and workflows**, which integrations you use and your recurring patterns
- **Context**, projects, repositories, channels you work in

When you start a new conversation, GAIA searches these memories and pulls in whatever is relevant to your request, automatically. Memory also drives GAIA's proactive behavior: the more it knows about your routines, the better it anticipates what you need.

<Tip>
  The more you use GAIA with your connected integrations, the richer your
  memory becomes. One conversation where you message "Sarah" on Slack teaches
  GAIA her Slack ID for good.
</Tip>

## Teach it directly

You can add memories yourself instead of waiting for GAIA to pick them up.

<Steps>
  <Step title="Open memory settings">
    Go to **Settings** and select **Memories** from the sidebar.
  </Step>
  <Step title="Add a memory">
    Click **Add Memory** and type what you want GAIA to know. Be specific,
    "My manager is Alex Chen, alex.chen@company.com" or "I prefer meeting
    agendas as bullet points."
  </Step>
  <Step title="Save">
    Click **Save Memory**. GAIA processes the text and pulls out the key facts.
  </Step>
</Steps>

<Note>
  GAIA won't store a memory if it can't extract something meaningful. "I like
  coffee" is too vague. "I order from Blue Bottle, size large, oat milk" gives
  GAIA something it can use.
</Note>

## See and control what it knows

The Memories page in Settings shows everything GAIA has learned, two ways:

- **Graph view**, a visual network of people, tools, and projects and how they connect. The default view. You can export it as a PNG or SVG.
- **List view**, every stored memory with its content, date, and category (identity mappings, preferences, context patterns).

<Frame>
  <img className="rounded-lg" src="/images/guides/memory/memory-list.png" alt="Memory list view showing stored memories with categories and dates" />
</Frame>

To remove something:

- **Delete one memory**, in List view, click the delete button next to it.
- **Clear all memories**, click **Clear All** to erase everything.

<Warning>
  Clearing all memories cannot be undone. GAIA resets its knowledge of your
  preferences, contacts, and workflows and re-learns from future conversations.
</Warning>

## How it works under the hood

You can skip this, memory works without you knowing any of it.

GAIA's memory runs on vector-based semantic search. Each memory is embedded as a vector, a numerical representation of its meaning, so recall doesn't depend on exact keywords. Asking about "my team lead" surfaces a memory stored as "Alex Chen is my manager," because the concepts are related.

Memories also carry graph relationships, structured connections between people, tools, and projects, so GAIA understands how things connect, not just isolated facts.

**Privacy:**

- Memories are per-user. No other user can access yours.
- They're processed and stored in a dedicated memory service, separate from your chat history.
- You can view, add, and delete any memory at any time from Settings.
